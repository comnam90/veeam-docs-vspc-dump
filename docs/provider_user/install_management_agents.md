# Deploying Veeam Service Provider Console Management Agents


To manage and monitor Veeam backup agents, Veeam Backup & Replication, Veeam ONE and Veeam Backup for Microsoft 365 in company infrastructure, you must install Veeam Service Provider Console management agents on machines where managed Veeam products are deployed and configure these agents to communicate with Veeam Service Provider Console. You can install management agents on Windows, Linux and macOS machines.

You can also use management agents installed on Windows and Linux machines as master agents for discovery in the company infrastructure.

|  |
| --- |
| Note: |
| You cannot install management agents on computers where Linux Veeam Backup & Replication server is deployed. For details on connecting Linux Veeam Backup & Replication server, see section [Connecting Veeam Backup & Replication Servers](connect_backup_servers.md). |

To deploy Veeam Service Provider Console management agents, you can use the following methods:

* [Install management agents with discovery rules](install_agents_discovery.md).
* [Install management agents manually](deploy_management_agents.md).
