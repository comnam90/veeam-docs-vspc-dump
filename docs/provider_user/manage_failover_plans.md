# Managing Failover Plans


In case of a disaster in the production site when all critical VMs go offline and a Veeam Backup & Replication server becomes inaccessible, you can perform full site failover using Veeam Service Provider Console.

In Veeam Service Provider Console, you can manage failover plans configured on managed Veeam Backup & Replication servers, including:

* Local failover plans that include regular VM replicas.

For details on local failover plans, see section [Failover Plans](https://helpcenter.veeam.com/docs/vbr/userguide/failover_plan.html?ver=13) of the Veeam Backup & Replication User Guide.

* Cloud failover plans that include cloud VM replicas.

For details on cloud failover plans, see section [Cloud Failover Plan](https://helpcenter.veeam.com/docs/backup/cloud/cloud_failover_plan_overview.html) of the Veeam Cloud Connect Guide.

To access failover plans, you must have at least one Veeam Backup & Replication server in your infrastructure.

You can start, test and undo failover plans, view and export failover plan details.
