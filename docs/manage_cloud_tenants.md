# Managing Cloud Tenants


In Veeam Service Provider Console, Veeam Cloud Connect tenant accounts with allocated cloud backup and replication resources are represented as cloud tenants. A cloud tenant is an account that consumes Veeam Cloud Connect backup and replication resources offered by a service provider, including:

* Deployment, configuration and management of Veeam backup agents on client computers.
* Management of Veeam Backup & Replication installations and jobs for a client.
* Hosting clients' backups and replicas created with Veeam Backup & Replication and Veeam backup agents on the service provider side.

To work with a cloud tenant in Veeam Service Provider Console, you must register an account for this cloud tenant.

Each cloud tenant that has a registered account in Veeam Service Provider Console can access the Client Portal to view invoices, manage Veeam backup agents and Veeam Backup & Replication jobs, and so on. For details on Veeam Service Provider Console features available to company users, see [Guide for End Users](https://helpcenter.veeam.com/docs/vac/provider_user/about.html).

In Veeam Service Provider Console plugin, you can perform the following cloud tenant management operations: create new cloud tenants, modify cloud tenants properties, delete cloud tenants, disable and enable cloud tenants, and map cloud tenants to client companies.
